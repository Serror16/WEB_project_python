## Инструкция по структуре и логике продукта
### Логика запроса
`Client -> API -> Service -> Adapter`


* __Client:__
    * __Отправляет__ `POST /api/v1/tax/report?country=russia` с JSON-телом.  

    * __Где:__ Внешний сервис или Postman/Swagger.

* __API Layer:__
    * __Файлы:__ `app/api/v1/endpoints/tax.py` и `app/schemas/tax.py`.

    * __Что происходит:__
        * FastAPI автоматически сопоставляет JSON с Pydantic-моделью в `schemas/tax.py`.  

        * Если данные невалидны (например, отрицательная сумма), API сразу вернет 422 ошибку.

        * Если всё ок, контроллер в `endpoints/tax.py` просто вызывает метод сервиса: `tax_service.send_report(data)`.

* __Service Layer:__
    * __Файлы:__ `app/services/tax_service.py`.

    * __Что происходит:__

        * __Маршрутизация:__ Сервис смотрит на параметр `country` и решает, какой адаптер вызвать.  

        * __Отказоустойчивость:__ Здесь находится Retry-логика. Если адаптер упал, сервис пробует еще раз.  

        * __Идемпотентность:__ Проверяет `idempotency-key`, чтобы не отправить один и тот же отчет дважды.  

        * __Логирование:__ Вызывает `app/db/models.py`, чтобы сохранить факт запроса к БД.  

* __Adapter Layer:__
    * __Файлы:__ `app/adapters/russia_adapter.py`.

    * __Что происходит:__

        * __Трансформация:__ Конвертирует чистый JSON в формат, который понимает внешняя налоговая (например, в XML через `app/utils/xml_parser.py`).  

        * __Специфика:__ Добавляет нужные заголовки аутентификации или SOAP-обертки.  

        * __Инкапсуляция:__ Если внешняя налоговая вернула странную ошибку в формате XML, адаптер ловит её и трансформирует в нашу Унифицированную модель ошибки (`app/schemas/errors.py`).  

* __External Service:__
    * __Файлы:__ Папка `mocks/`.

    * __Что происходит:__
        * Имитирует реальную налоговую. Один мок отвечает быстро по REST, другой — медленно по SOAP, третий специально "отваливается", чтобы проверить Retry-логику в сервисе.
